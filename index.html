GLM Matrix Functions Redesign

Intro:

The website I chose for this project is the GLM matrix functions documentation page. GLM is the library students use for the introductory Computer Graphics course at Brown, so it is fairly popular. I chose this site because of this popularity as well as its many flaws. Redesigning this site could lead to me making an actual impact on the Brown community.

https://glm.g-truc.net/0.9.4/api/a00133.html

Analyzing Problems:

  • The most glaring issue with this site is its small font size, which limits readability and accessibility and makes it harder for users to find the specific information they’re looking for.
  • Many of the functions on this page are also documented redundantly, which leads to the same kind of information taking up unnecessary space. For example, there are three different versions of the determinant function listed, one for a 2x2 matrix, a 3x3 matrix, and a 4x4 matrix respectively.
  • One more information-related issue is the spatial separation between the declaration of a function and its actual description. It would be simpler for users if the description was in the same area as its declaration.
  • The “functions” hyperlink on the far top right of the page, which takes you to the function description section, is redundant.
  • The “GLM Core” hyperlink is awkwardly positioned, and is important enough to simply include as part of the navigation options at the top of the page
  • The background image is visually complex, which can distract users from the actual content. Information about the functions should be the main priority and this should be reflected in aesthetic design choices.
  • The choice of orange and white not only makes it difficult to distinguish function descriptions from the background of the container div, but also gives off a feeling of heat, which I personally found to increase my stress while working on computer graphics projects (which is further exacerbated by the smoky background image). If I’m looking for complex information that I do not fully understand, I want to encounter that information in an environment that calms me down rather than heating me up.
  • Aria: The WebAIM WAVE detected thirty instances of low contrast, which I completely agree with. Besides that, it also gave thirty-two alerts related to layout elements, but did not specify them. Lastly, it detected that the site completely lacks ARIA landmarks
